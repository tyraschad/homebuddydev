# HomeBuddy Design References

Pinned screenshots of the source-of-truth designs. When you say a **short name** below in chat (e.g. *"match the Onboarding Data layout"* or *"use Basic Button styling"*), I open the matching entry here and follow the image.

Related: [brand-guidelines.md](../brand-guidelines.md) for tokens (colors, fonts, logos). Drift report: [onboarding-audit.md](../onboarding-audit.md).

## How to reference these

- Say the **short name** in a sentence — names are stable across sessions, uploads aren't.
- Reference **one thing per instruction** for the cleanest signal.
- To update a reference, upload a new version and say *"replace the X reference"* — the pointer file gets overwritten.
- To annotate a reference with a rule, add a note under its row here — I read those notes.
